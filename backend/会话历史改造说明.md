# 会话历史（历史对话）改造说明

本文说明后端如何持久化「用户消息 + 助手回复」、**会话元问题**如何短路编排，以及总结器行为。

## 1. 数据从哪来、何时写入

- 会话状态在 [`app/core/state.py`](app/core/state.py) 的 `ConversationState` 中，字段为 **`history: list[ConversationTurn]`**（`ConversationTurn` 含 `role`、`content`、`intent`）。
- 每一轮图跑完后，在 [`app/core/orchestrator.py`](app/core/orchestrator.py) 的 **`_save_state_node`** 里调用：
  - `session_store.append_history(..., "user", state["text"], ...)`
  - `session_store.append_history(..., "assistant", state["result"].message, ...)`
- 因此：**本轮用户输入在 `save_state` 之前尚未进入 `history`**；若用户问「我刚才说了什么」，元问题路径会结合 **已落库 history + 本轮 `text`**（见第 3 节）。

## 2. 历史长度

- 配置项 **`MAX_HISTORY_TURNS`**（[`app/core/settings.py`](app/core/settings.py)，默认约 20）表示「轮次」量级。
- **`append_history` 不再做滑动截断**，`history` 仅追加、由部署侧或后续策略控制总量。
- **元问题回复格式化**：在 [`app/core/session_meta.py`](app/core/session_meta.py) 的 **`format_meta_session_reply`** 中，从 `history` **尾部**取约 **`max(2, MAX_HISTORY_TURNS * 2)` 条**消息窗口，并**突出列出其中的用户发言**（编号列表）。

## 3. 会话元问题（编排短路）

- 在 [`app/core/session_meta.py`](app/core/session_meta.py) 中 **`is_session_meta_question`** 做规则匹配（如「刚才问了什么」「会话历史」等）。
- [`app/core/orchestrator.py`](app/core/orchestrator.py)：`load_state` 之后 **条件边** → **`session_recall`** 或 **`decompose`**。
- **`session_recall`**：根据 `conversation.history` 生成纯文本回复，**不经过** `decompose`、SQL、RAG 等节点；边为 **`session_recall` → `collect_result` → `dispatch` → … → `summarize` → `save_state`**。
- 子任务意图为 **`session_meta`**；总结器对该意图 **直接输出已生成文案**，不注入全会话历史、不额外走历史 LLM。

## 4. 总结器（query / rule / 其他）

- [`app/agents/summarizer_agent.py`](app/agents/summarizer_agent.py)：**不再**使用模块级「全会话历史块」注入 query/rule；检索总结仅基于 **本轮用户输入、子任务原文、引用证据** 等。
- **order / handoff**：仍为固定模板字符串。
- **unknown / else**：使用子任务返回的 `message`，不套历史 LLM。

## 5. 相关文件一览

| 文件 | 作用 |
|------|------|
| [`app/core/session_meta.py`](app/core/session_meta.py) | 元问题判定、`format_meta_session_reply` |
| [`app/agents/summarizer_agent.py`](app/agents/summarizer_agent.py) | 按子任务总结；`session_meta` 原样输出 |
| [`app/core/session_store.py`](app/core/session_store.py) | `append_history`（仅追加） |
| [`app/core/orchestrator.py`](app/core/orchestrator.py) | `session_recall` 节点与条件边、`_save_state_node` |
| [`app/core/settings.py`](app/core/settings.py) | `MAX_HISTORY_TURNS` |

## 6. 使用注意

- 首轮会话 **`history` 为空**时，元问题回复会说明尚无更早记录，属预期。
- 依赖 **同一 `session_id`** 复用会话；换会话 ID 则没有上一轮的记录。
